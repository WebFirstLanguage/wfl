About Nexus

Nexus is a lightweight, semantic server framework built entirely in WebFirst Language (WFL). It provides a clean, understandable architecture for building web applications.

The Philosophy

Nexus follows the principle of "lazy routing" - it tries the easiest thing first (serving a static file) before doing any real work (running a handler). This makes it fast and efficient by default.

Architecture

The framework uses a strict Server -> Router architecture: 

Server (The "Bouncer")
  - Listens for connections
  - Passes ALL requests to Router
  - No business logic here

Router (The "Traffic Cop")
  - Check 1: Static file? Serve it!
  - Check 2: Dynamic route? Run handler!
  - Check 3: Nothing? Return 404

Handler (The "Talent")
  - Receives request object
  - Returns response

Usage

To add a new route, simply register it and define a handler: 

// Register the route
route_get with "/my-page" and "handle_my_page"

// Define the handler
define action called handle_my_page needs request:
    respond_json with request and "{\"message\": \"Hello!\"}"
end action

This page itself is served as a static file. Try accessing /about (without .html) - the router automatically resolves it!
