# WFL Documentation Hub

Welcome to the complete documentation for the WebFirst Language (WFL)! This is your starting point for learning WFL, from your first "Hello, World!" to building production web servers.

> **Programming that reads like plain English**

> 🧪 **The code in these docs is tested.** Every runnable `wfl` example is executed
> against the release binary by [`scripts/test_docs_code_blocks.py`](../scripts/test_docs_code_blocks.py)
> — placeholder templates, illustrative fragments, and long-running server demos
> are classified rather than run; the current state is tracked in
> [`DOC_CODE_AUDIT.md`](../TestPrograms/docs_examples/DOC_CODE_AUDIT.md).
> New here? Start with the output-verified [Friendly WFL Tour](../TestPrograms/docs_examples/tour/README.md).

## 🔍 Quick Reference

### Common Syntax

```wfl
// Variables
store name as "Alice"
change name to "Bob"

// Display
display "Hello, World!"
display "Value: " with variable

// Conditionals
check if condition:
    // code
otherwise:
    // code
end check

// Loops
count from 1 to 10:
    display count
end count

for each item in list:
    display item
end for

// Actions
define action called name with parameters x and y:
    return x plus y
end action

call name with 5 and 3

// Lists
create list items:
    add "first"
    add "second"
end list

push with items and "third"

// Error Handling
try:
    // risky code
catch:
    // handle error
end try
```

### Common Built-in Functions

```wfl
// Core
display "text"
typeof of value
isnothing of value

// Math
abs of -5          // 5
round of 3.7       // 4
floor of 3.7       // 3
ceil of 3.2        // 4

// Text
touppercase of "hello"      // "HELLO"
tolowercase of "HELLO"      // "hello"
length of "hello"           // 5
substring of "hello" from 0 length 2  // "he"

// Lists
length of list
push with list and item
pop from list
```

## 🌟 Highlights

### Why WFL?

```wfl
// Code that reads like English
for each customer in premium_customers:
    check if customer_balance is greater than 1000:
        send_loyalty_reward to customer
    end check
end for
```

**[Learn why natural language syntax matters →](01-introduction/why-wfl.md)**

### Built-in Web Server

```wfl
listen on port 8080 as server

wait for request comes in on server as req
respond to req with "Hello from WFL!"
```

**No frameworks needed!** Web servers are built-in.

### Type Safety

```wfl
store age as 25
store name as "Alice"

// WFL catches this error before running:
// display age plus name  // ERROR: Cannot add Number and Text
```

**[Learn about type safety →](03-language-basics/variables-and-types.md#type-safety)**
